# Dreamforce 2026 Staffing Optimization

**Project Owner:** Summer Hash  
**Project Advisor:** You (previous staffing lead)  
**Event Dates:** September 15-17, 2026  
**Project Start:** May 2026

## Overview

This project takes an AI-first approach to optimize and automate Slack's staffing operations for Dreamforce 2026. Rather than simply documenting last year's manual process, we're identifying jobs-to-be-done (JTBD) and determining where automation can reduce manual effort and improve efficiency.

## Goals

1. **Map all jobs-to-be-done** for Dreamforce staffing lead role
2. **Identify automation opportunities** to reduce manual hours
3. **Create tools and dashboards** to give Summer visibility and control
4. **Document the optimized process** for future events
5. **Reduce last-minute chaos** through better systems and workflows

## Problem Statement

Dreamforce staffing currently involves:
- Multiple disconnected systems (Rainfocus, Staffing Force, spreadsheets, Google Calendar)
- Unclear source of truth for event footprint, sessions, and requirements
- Manual tracking of 3-4 stage process (apply в†’ approve в†’ assign pass в†’ schedule shifts)
- "Horror show" of last-minute changes with no self-service capability
- Significant time spent reconciling data across systems

## Key Deliverables

- Jobs-to-be-done analysis with automation opportunities
- Process documentation and timeline dashboard
- Data integration and visibility tools
- Self-service capabilities for common requests
- Handoff materials for Summer Hash
